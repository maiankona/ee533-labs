Streaming Inference Pipeline Plan

Pipelined full-duplex architecture: input packets arrive continuously from nf7, inference runs on GPU, result packets stream back — all in hardware without host software intervention.

Four-Stage Pipeline
Stage 1
Packet RX FSM
ids.v
Receives incoming packet on nf2c3, strips IOQ header, writes payload words to GPU Inputs region (0x10–0xAF). Asserts in_rdy=0 while GPU busy.
Stage 2
ARM Polling Loop
pipeline_pseudoARM
Polls packet_word_count at 0x00. On nonzero, dispatches GPU via STR to 0xF0–0xF2. Clears flag after dispatch.
Stage 3
GPU Inference
pipeline_backup
Runs BF16 tensor inference (VMAC + ReLU) on inputs from 0x10+. Writes outputs to 0xD0–0xEF. Asserts pipeline_done on completion.
Stage 4
Packet TX FSM
ids.v
Detects pipeline_done, reads outputs from 0xD0–0xEF, constructs NF2.1 packet, drives net_tx_data/valid back to nf7. Clears 0x00 to unblock RX.
Pipeline Handshake Sequence (Single Packet)
RX FSM
Packet arrives on nf2c3 — detect IOQ header (in_ctrl != 0), skip it. Begin writing in_data words to BRAM starting at 0x10.
RX FSM
EOP detected (in_ctrl != 0 again) — write word count to 0x00 (packet_word_count). Assert in_rdy=0 to stall upstream.
packet_word_count ← N
ARM
Polling loop detects 0x00 != 0. Loads word count into R2. Writes cfg_inst_addr, cfg_length, gpu_dispatch to 0xF0–0xF2.
gpu_dispatch fired
GPU
Fetches instructions from IMEM, loads inputs via LW from 0x10+, loads weights via LW from 0xB0+, runs VMAC + ReLU, writes outputs via ST to 0xD0+.
pipeline_done ← 1
TX FSM
Detects pipeline_done. Reads output words from 0xD0–0xEF. Constructs NF2.1 packet (IOQ header + payload). Drives net_tx_data/valid.
result packet → nf7
TX FSM
Packet transmission complete. Clear packet_word_count at 0x00 to zero. Deassert in_rdy=0 — RX FSM now ready for next packet.
packet_word_count ← 0

Design Constraints & Decisions

Buffering Strategy

  • Single buffer — stall on busy (proof of concept)
  • RX asserts in_rdy=0 while GPU running
  • nf7 must wait or buffer — controlled by us
  • Upgrade to ping-pong after PoC verified

Data Format

  • BF16 packed as two values per 64-bit word
  • Inputs arrive in packet payload, fixed layout
  • Weights preloaded once by Perl host script
  • Output packet format: raw BF16 output words

Timing

  • 125 MHz datapath clock
  • 2-2-1 ANN inference ≈ 100–200 cycles
  • Packet TX must complete before next RX
  • ARM dispatch overhead ≈ 25 cycles

Open Questions

  • Result packet header format — sequence number?
  • Who drives net_tx — TX FSM or FIFO mode?
  • Error handling — what if GPU times out?
  • How does nf7 know result corresponds to input N?
