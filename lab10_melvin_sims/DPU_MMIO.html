ARM Control Region Detail — 0x00–0x0F
Address Name Direction Description
0x00 packet_word_count FSM → ARM Number of payload words written by packet FSM on EOP. Zero = not ready. ARM polls this, clears after GPU dispatch.
0x01–0x0F — — Reserved for future ARM control registers
GPU Control Register Detail — 0xF0–0xFF
Address Name Direction Description
0xF0 cfg_inst_addr ARM → GPU Base address of GPU program in IMEM window
0xF1 cfg_length ARM → GPU Number of instructions to fetch (max 64)
0xF2 gpu_dispatch ARM → GPU Write any value to trigger FSM IDLE → EXECUTING
0xF3 arm_irq_ack ARM → GPU Write any value to trigger FSM DONE → IDLE, clears gpu_irq
0xF4–0xFF — — Reserved for future control registers
Packet FSM Behavior
State Trigger Action
IDLE in_wr && in_ctrl != 0 IOQ module header detected — skip word, move to CAPTURE
CAPTURE in_ctrl == 0 Write in_data to BRAM at 0x10 + word_ptr, increment pointer
EOP in_ctrl != 0 Write word_ptr to 0x00 (packet_word_count), reset pointer, return to IDLE
DN-ASIC · EE533 · USC Viterbi · BRAM: 256 × 64-bit words on NetFPGA XC2VP50 (convertible_fifo / gpu_fifo_mem). GPU IMEM loaded directly — no BRAM window required. Weights preloaded once per session via host Perl scripts. Packet payload arrives via nf2c3 (10.0.15.2) from node3 (10.0.15.3).
